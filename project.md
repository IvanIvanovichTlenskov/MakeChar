# Спецификация проекта: MakeChar (Stable Diffusion Plugin)

## 1. Обзор
**MakeChar** — это плагин для веб-интерфейса Stable Diffusion (например, Automatic1111 или Forge). Плагин добавляет новую вкладку в интерфейс, предназначенную для удобного создания и сохранения профилей персонажей. Основная цель — объединить визуальные референсы и текстовые описания в единый промпт.

## 2. Интерфейс Пользователя (UI)

Плагин добавляет вкладку с названием **"MakeChar"**. Интерфейс разделен на три логические зоны:

### 2.1. Верхняя панель (Управление персонажами)
Расположена в самом верху вкладки.
*   **Выпадающий список (Dropdown):** Отображает список сохраненных персонажей.
*   **Кнопка "Создать нового" (New):** Очищает поля для создания нового персонажа.
*   **Кнопка "Сохранить" (Save):** Сохраняет текущее состояние (картинки и текст) на жесткий диск под выбранным именем или создает новый файл.

### 2.2. Основная зона (Конструктор персонажа)
Состоит из трех горизонтальных рядов (слотов). Каждый ряд содержит:
*   **Слева:** Поле для изображения (`Image Upload`).
    *   *Функционал:* Поддержка Drag & Drop и **вставки из буфера обмена (Ctrl+V / Cmd+V)**.
*   **Справа:** Текстовое поле (`Text Input`).
    *   *Назначение:* Ввод описания атрибута (например, "прическа", "одежда", "аксессуары").

**Схема ряда:**
`[ Изображение 1 ]`  `[ Текст 1 ]`
`[ Изображение 2 ]`  `[ Текст 2 ]`
`[ Изображение 3 ]`  `[ Текст 3 ]`

### 2.3. Нижняя панель (Агрегатор промпта)
Расположена под тремя рядами.
*   **Текстовое поле (Read-only или Editable):** Автоматически объединяет текст из трех верхних полей через пробел.
    *   *Логика:* `Text_Final = Text_1 + " " + Text_2 + " " + Text_3`
*   **Кнопка "Send to Positive Prompt" (Справа от поля):**
    *   При нажатии содержимое нижнего текстового поля копируется в основное поле "Positive Prompt" главной вкладки генерации (txt2img / img2img).

## 3. Функциональные требования

### 3.1. Работа с изображениями
1.  Пользователь может вставить изображение прямо в поле ввода из буфера обмена.
2.  При сохранении персонажа изображения должны сохраняться в локальную папку проекта (например, `./extensions/MakeChar/characters/[char_name]/`).
3.  При загрузке персонажа изображения должны подгружаться из сохраненной папки в соответствующие поля.

### 3.2. Работа с текстом
1.  Изменение текста в любом из трех верхних полей должно мгновенно обновлять содержимое нижнего (четвертого) поля.
2.  Нижнее поле является результатом конкатенации.

### 3.3. Сохранение и Загрузка (Persistence)
1.  **Формат хранения:** Данные сохраняются в формате JSON.
2.  **Структура данных:**
    ```json
    {
      "character_name": "Name_String",
      "slots": [
        {
          "image_path": "relative/path/to/image1.png",
          "text": "описание атрибута 1"
        },
        {
          "image_path": "relative/path/to/image2.png",
          "text": "описание атрибута 2"
        },
        {
          "image_path": "relative/path/to/image3.png",
          "text": "описание атрибута 3"
        }
      ]
    }
    ```
3.  **Список персонажей:** При старте плагина сканирует папку сохранения и заполняет верхний выпадающий список именами файлов/персонажей.
4.  **Выбор персонажа:** При выборе имени из списка, интерфейс очищается и заполняется данными из соответствующего JSON файла и картинок.

### 3.4. Интеграция с SD WebUI
1.  Кнопка отправки промпта должна использовать JavaScript для поиска DOM-элемента основного положительного промпта и обновления его значения (`textarea` с id `txt2img_prompt` или аналогичным).

## 4. Техническая реализация (Рекомендации)

*   **Фреймворк:** Gradio (стандарт для SD WebUI).
*   **Backend (Python):**
    *   Обработка путей к файлам.
    *   Сохранение изображений (Pillow).
    *   Сериализация/Десериализация JSON.
*   **Frontend (JS/CSS):**
    *   Обработчик события `paste` для полей изображения.
    *   Обработчик события `click` для кнопки отправки промпта (взаимодействие с глобальной областью видимости WebUI).
    *   CSS для выравнивания рядов (Flexbox или Grid), чтобы картинка и текст были на одной линии.

## 5. Пользовательский сценарий (User Flow)

1.  Пользователь открывает вкладку **MakeChar**.
2.  Нажимает **"Создать нового"**.
3.  Копирует картинку прически в браузере, кликает в первое поле картинки и нажимает `Ctrl+V`. Вводит текст "Red long hair".
4.  Повторяет для одежды и аксессуаров.
5.  Видит итоговый промпт снизу: "Red long hair Blue dress Silver necklace".
6.  Нажимает кнопку отправки -> Текст появляется в главном окне генерации.
7.  Вводит имя "Warrior Girl" и нажимает **"Сохранить"**.
8.  В следующий раз выбирает "Warrior Girl" из списка, и все поля заполняются автоматически.
